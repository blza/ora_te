Method Detail

ASSERT

  
ASSERT(A IN BOOLEAN)
Accert procedure. Is to be called from unit test functions. Automatically gets caller info and stores
it and the result of accertion in ut_report table. As all unit tests are Package body procedures their name can only
be infered during analysis of package body source code. This analysis is done when accessing ut_report_deciphered view
Parameters:
A - accertion as a boolean value. If accertion is true, 'Passed' is inserted in ut_report along with caller info, 'Failed' otherwise.

GET_CALLER_INFO

  
GET_CALLER_INFO(A_DESIRED_DEPTH IN PLS_INTEGER, 
                  A_REAL_DEPTH OUT PLS_INTEGER, 
                  A_CALLER_TYPE OUT VARCHAR2, 
                  A_CALLER_OWNER OUT VARCHAR2, 
                  A_CALLER_NAME OUT VARCHAR2, 
                  A_LINE_NUMBER OUT PLS_INTEGER)
Gets calling module info including owner, type, name, line at desired call stack depth The refactored version of one that Tom Kyte wrote
Parameters:
A_DESIRED_DEPTH - the depth in the call stack from this function and up
So say providing 2 as a depth we can get not the caller of GET_CALLER_INFO but the caller of the function that called GET_CALLER_INFO
A_REAL_DEPTH - returns the real depth if it's less then desired
A_CALLER_TYPE - returns caller type ('PACKAGE', 'PACKAGE BODY', etc.)
A_CALLER_OWNER - returns caller owner
A_CALLER_NAME - returns caller name
A_LINE_NUMBER - returns caller line number
